# TDengine-Industrial-Data-Pipeline

[中文](./README.md) | [English](./README_EN.md)

`TDengine-Industrial-Data-Pipeline` 是一个面向工业时序场景的数据工程工具链，覆盖 TDengine 建表、数据写入、时序切分、时空表构建以及训练数据导出。

## 项目定位

工业时序数据往往存在这些特点：

- 数据源杂乱
- 字段不统一
- 时间粒度复杂
- 需要同时保留时间特征和空间特征
- 最终还要服务下游模型训练和诊断任务

这个项目的目标，是把“原始工业数据 -> TDengine -> 特征表 -> SFT 训练数据”的链路系统化。

## 当前能力

- TDengine 表结构创建
- 工业数据写入 TDengine
- 时间表、空间表与时空表构建
- 特征加工和序列切分
- 导出 JSON / SFT 训练数据

## 这个项目是为了解决什么

工业时序数据真正难的地方，往往不是“有没有数据”，而是“这些数据能不能被下游系统稳定使用”。

- 原始数据字段不统一
- 时间粒度和采样方式不稳定
- 空间关系和时间关系经常混在一起
- 想做训练样本时，还需要额外再整理一遍

这个项目就是希望把原始工业数据一步步整理成可入库、可分析、可训练的结构化数据资产。

## 如何使用这个项目

一个典型思路是：

1. 先把原始工业时序数据接入并写入 TDengine。
2. 根据业务需要构造时间表、空间表和时空表。
3. 在表级别做特征清洗、切片和聚合。
4. 最后把可用数据导出成 JSON 或 SFT 训练样本，供大模型或诊断模型继续使用。

它适合：

- 工业数据治理项目
- 设备监测和故障诊断任务
- 需要把原始时序数据转成训练语料的团队
- 想把数据库链路和模型链路打通的场景

## 这个项目大概是怎么工作的

当前的设计逻辑可以粗略分成四步：

- 入库
  - 先定义 TDengine 表结构，把原始数据稳定写进去
- 结构化整理
  - 把原始数据切成时间表、空间表或二者结合的时空表
- 特征加工
  - 在表结构之上做清洗、切片、聚合和上下文拼接
- 样本导出
  - 把结构化结果转成训练 JSON 或 SFT 样本格式

也就是说，这个项目连接的是“数据库工程”和“模型数据工程”之间的那一段中间层。

## 它能给大家带来什么帮助

- 帮工业团队把脏乱的原始数据变成可复用的数据底座
- 帮算法团队减少手工清洗和反复转格式的工作
- 帮项目更快搭起从原始数据到模型训练样本的链路
- 帮数据库侧和模型侧之间建立更稳定的协作接口
