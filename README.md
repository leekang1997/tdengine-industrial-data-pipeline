# TDengine-Industrial-Data-Pipeline

[中文](./README.md) | [English](./README_EN.md)

`TDengine-Industrial-Data-Pipeline` 是一个面向工业时序场景的数据工程工具链，覆盖 TDengine 建表、数据写入、时序切分、时空表构建以及训练数据导出。

## 一眼看懂

- 项目类型：建立在 `TDengine` 之上的数据治理与样本构建层
- 核心目标：把原始工业时序数据整理成可分析、可训练的数据资产
- 适合人群：做工业数据治理、故障诊断和训练样本构建的团队
- 依赖前提：必须先安装并运行好 `TDengine`
- 当前阶段：开源初版，重点展示数据链路组织与样本构建思路

## 项目定位

工业时序数据往往存在这些特点：

- 数据源杂乱
- 字段不统一
- 时间粒度复杂
- 需要同时保留时间特征和空间特征
- 最终还要服务下游模型训练和诊断任务

这个项目的目标，是把“原始工业数据 -> TDengine -> 特征表 -> SFT 训练数据”的链路系统化。

## 当前能力

- TDengine 表结构创建
- 工业数据写入 TDengine
- 时间表、空间表与时空表构建
- 特征加工和序列切分
- 导出 JSON / SFT 训练数据

## 使用前提

这个仓库不是独立替代 `TDengine` 的数据库系统，而是建立在 `TDengine` 之上的数据治理与样本构建流程层。

在使用之前，你至少需要：

1. 已经安装并启动好 `TDengine`
2. 有权限连接目标数据库
3. 明确原始数据的字段结构、时间粒度和采样方式
4. 明确你要构建的是时间表、空间表、时空表，还是训练样本导出链路

如果没有先准备好数据库环境和原始数据，这个仓库不能单独完成数据链路搭建。

## 这个项目是为了解决什么

工业时序数据真正难的地方，往往不是“有没有数据”，而是“这些数据能不能被下游系统稳定使用”。

- 原始数据字段不统一
- 时间粒度和采样方式不稳定
- 空间关系和时间关系经常混在一起
- 想做训练样本时，还需要额外再整理一遍

这个项目就是希望把原始工业数据一步步整理成可入库、可分析、可训练的结构化数据资产。

## 如何使用这个项目

一个典型思路是：

1. 先安装并验证 `TDengine`。
2. 准备好原始工业时序数据和字段映射关系。
3. 把原始数据接入并写入 TDengine。
4. 根据业务需要构造时间表、空间表和时空表。
5. 在表级别做特征清洗、切片和聚合。
6. 最后把可用数据导出成 JSON 或 SFT 训练样本，供大模型或诊断模型继续使用。

它适合：

- 工业数据治理项目
- 设备监测和故障诊断任务
- 需要把原始时序数据转成训练语料的团队
- 想把数据库链路和模型链路打通的场景

## 这个项目大概是怎么工作的

当前的设计逻辑可以粗略分成四步：

- 入库
  - 先定义 TDengine 表结构，把原始数据稳定写进去
- 结构化整理
  - 把原始数据切成时间表、空间表或二者结合的时空表
- 特征加工
  - 在表结构之上做清洗、切片、聚合和上下文拼接
- 样本导出
  - 把结构化结果转成训练 JSON 或 SFT 样本格式

也就是说，这个项目连接的是“数据库工程”和“模型数据工程”之间的那一段中间层。

## 你需要配置哪些内容

至少需要明确下面这些配置：

- TDengine 连接信息
  - 例如主机、端口、数据库名、账号权限
- 原始数据字段映射
  - 明确哪些字段是时间、标签、数值、设备标识等
- 表结构设计
  - 需要哪些时间表、空间表、时空表
- 数据导出目标
  - 最终是要导出 JSON、训练样本，还是中间特征结果

最关键的一点是：

- 这个项目负责“围绕 TDengine 做数据整理、特征构造和样本导出”
- `TDengine` 负责“真正存储和查询原始时序数据”

所以如果底层数据库环境没有准备好，数据治理流程也无法真正落地。
